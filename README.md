## 快速开始

安装 Skill 后，在 AI Agent 中直接用自然语言描述生图需求即可：

```
用户: 帮我画一只橘猫趴在窗台上晒太阳
→ Skill 自动触发 → 推荐模型 → 优化提示词 → 生成图片 → 保存
```

**你不需要**：
- 手动选模型（Skill 会根据场景自动推荐）
- 写专业提示词（Skill 会自动增强）
- 记 API 参数（Skill 自动处理尺寸、格式等）

**你需要做的**：
- 说清楚你想要什么（主体、场景、风格）
- 看结果后告诉 Skill 满不满意

## 环境准备

### 1. API Key

从 [阿里云百炼控制台](https://dashscope.console.aliyun.com/) 获取 API Key，然后配置到环境中：

#### macOS / Linux

在 `~/.zshrc`（或 `~/.bashrc`）中添加：

```bash
export Qwen_DASHSCOPE_API_KEY="sk-你的密钥"
```

执行 `source ~/.zshrc` 使其生效。

#### Windows (PowerShell)

在当前会话临时设置：

```powershell
$env:Qwen_DASHSCOPE_API_KEY = "sk-你的密钥"
```

永久设置（写入系统环境变量）：

```powershell
[System.Environment]::SetEnvironmentVariable("Qwen_DASHSCOPE_API_KEY", "sk-你的密钥", "User")
```

然后重启 PowerShell 使其生效。

#### 最简单的方式：让 Agent 帮你配置

如果你不确定怎么操作，直接告诉 Agent：

```
帮我配置 Qwen_DASHSCOPE_API_KEY，我的 Key 是 sk-xxxxxxxx
```

Agent 会自动检测你的操作系统并写入对应的配置文件。

### 2. Python 依赖

Skill 依赖以下 Python 包：

```
dashscope, requests, pillow
```

**正常情况下不需要你手动安装。** 当 Skill 首次触发时，Agent 会自动检测并安装缺失的依赖。

如果自动安装失败（例如网络问题），可以手动让 Agent 帮你处理：

```
帮我安装 qwen-image-generator Skill 需要的 Python 依赖
```

或者自己执行：

```bash
pip install -r requirements.txt
# 或手动安装：pip install dashscope requests pillow
```

### 3. 验证

```bash
echo $Qwen_DASHSCOPE_API_KEY  # 应显示你的 Key
python3 -c "import dashscope; print('dashscope OK')"
```

## Skill 如何工作

```
你的自然语言描述
      ↓
  ① 意图理解 — 分析你想要什么（文生图？图生图？组图？）
      ↓
  ② 模型推荐 — 根据场景自动选择最佳模型
      ↓
  ③ 提示词处理 — 诊断完整度 → 增强缺失要素 → 适配模型风格
      ↓
  ④ 生成图片 — 调用 API → 下载图片
      ↓
  ⑤ 保存 & 展示 — 命名规范化 → 保存到 output/ → 展示给你
```

**整个过程中 Skill 会告诉你**：
- 推荐了哪个模型、为什么
- 提示词做了什么调整
- 生成的图片保存在哪里

## 写好提示词的关键

### 五个维度

Skill 会从五个维度评估你的提示词。越完整，生成效果越好：

| 维度 | 说明 | 示例 |
|------|------|------|
| **主体** | 画什么？ | 一只橘猫 |
| **场景** | 在哪里？ | 阳光洒满的窗台 |
| **光线** | 什么光线/氛围？ | 温暖的午后阳光 |
| **风格** | 什么调性？ | 治愈系水彩风 |
| **构图** | 怎么拍/怎么看？ | 特写、正面视角 |

### 提示词质量对比

```
太简单（完整度 20%）
"一只猫"
→ Skill 会自动增强，但结果可能不是你最想要的

还行（完整度 60%）
"一只橘猫趴在窗台上"
→ 有主体和场景，但缺少光线和风格

完整（完整度 85%）
"一只橘猫趴在阳光洒满的窗台上晒太阳，温暖的午后光线，治愈系风格，特写"
→ Skill 只做风格适配，不再大幅修改

非常完整（完整度 95%）
"一只毛茸茸的橘猫慵懒地趴在木质窗台上，窗外是樱花盛开的春天，
温暖的午后阳光从窗户洒进来，猫咪闭着眼睛微微打呼噜，
旁边放着一杯冒着热气的拿铁，治愈系日系插画风格，特写视角"
```

### 控制提示词优化

```
# 让 Skill 自由发挥（默认）
"帮我画一只猫"

# 明确要求优化
"帮我优化一下提示词，画一只赛博朋克风格的猫"

# 明确要求不优化（Skill 会跳过增强，只做风格适配）
"不要优化提示词，直接使用：一只橘猫趴在窗台上晒太阳"
```

## 组图生成指南

> **这是本 Skill 最强大的功能之一，也是最容易被误用的功能。**

### 什么是组图？

组图是用 **wan2.7 的顺序生成** 功能，一次生成多张风格一致、内容有连续性的图片。适合：
- 连环画 / 绘本
- 产品使用流程
- 人物动作序列
- 同一主题的系列图

### 三个核心触发词

| 关键词 | 作用 | 示例 |
|--------|------|------|
| **"组图"** | 告诉 Skill 要生成多张图 | "画一组小猫日常的组图" |
| **"连环画"** | 同上，暗示顺序生成 | "画一个连环画，讲小猫的故事" |
| **"故事"** | 触发 wan2.7 的故事生成模式 | "画一个小猫寻找主人的故事" |

### 需不需要指定张数？

**建议指定，但不是必须。**

| 你的说法 | Skill 的处理 |
|---------|------------|
| "画一组连环画"（不说张数） | 默认生成 4 张（wan2.7 story 模式） |
| "画 3 张连环画" | 生成 3 张 |
| "画一个 6 场景的故事" | 生成 6 张（注意：wan2.7 最多一次 4 张，会分批生成） |

> wan2.7 单次最多生成 **4 张**。如果需要更多，Skill 会分批生成。

### 组图提示词写法

#### 模板

```
用 wan2.7 画一组 [主题] 的 [组图/连环画/故事]，[N] 张，[尺寸偏好]

具体描述：
- 场景 1：xxx
- 场景 2：xxx
- 场景 3：xxx
（可选，Skill 也可以自动生成场景）
```

#### 示例 1：四季故事组图

```
用 wan2.7 画一组小猫的四季故事组图，4 张，竖版

一只可爱的小橘猫经历了春夏秋冬四个季节：
- 春天在樱花树下打滚
- 夏天在溪边玩水
- 秋天在落叶堆里玩耍
- 冬天在雪地里留下小脚印

每张图保持同一只橘猫、同一画风，治愈系绘本风格。
```

**Skill 的处理**：
1. 识别到 "组图" + "wan2.7" → 使用 wan2.7 顺序生成
2. 4 张 → n=4，启用 enable_sequential=True
3. 竖版 → size="portrait"（768×1152）
4. 提示词增强 + 自然中文风格适配

#### 示例 2：产品使用流程

```
帮我画一组咖啡机使用教程的连环画，4 张，横版

展示一台全自动咖啡机的操作步骤：
1. 打开盖子放入咖啡豆
2. 选择研磨模式，咖啡豆被磨成粉
3. 按下冲泡按钮，热水流过咖啡粉
4. 一杯香浓的咖啡倒进杯子里

产品渲染风格，背景简洁，重点突出操作动作。
```

**Skill 的处理**：
- "连环画" + "4 张" → wan2.7，n=4，顺序生成
- "横版" → size="landscape"（1152×768）

#### 示例 3：人物动作序列

```
画一个女孩做瑜伽的动作组图，4 张，方形

一个穿白色运动服的女孩在户外草地上做瑜伽：
- 第一个动作：山式站立，双手合十
- 第二个动作：前屈弯腰，手触脚尖
- 第三个动作：战士二式，双臂展开
- 第四个动作：树式平衡，单腿站立

同一人物、同一服装、同一场景，只是动作不同。
```

#### 示例 4：不指定场景（让 AI 自动编排）

```
用 wan2.7 画一个小狗成长的故事，4 张

一只金毛犬从幼犬到成犬的成长过程。
```

**Skill 会自动编排 4 个场景**：
- 场景 1：刚出生的小金毛，毛茸茸的
- 场景 2：几个月大，在草地上学走路
- 场景 3：半岁，开始调皮捣蛋
- 场景 4：成犬，稳重帅气

#### 示例 5：旅行主题绘本

```
帮我画一本小兔子的环球旅行绘本，4 张，竖版高清

小兔子背着小包去了四个国家旅行：
- 在法国巴黎的埃菲尔铁塔前拍照
- 在日本东京的樱花树下吃便当
- 在埃及的金字塔前骑骆驼
- 在美国纽约的自由女神像旁吃热狗

每张小兔子保持一样的造型，绘本插画风格。
```

#### 示例 6：负向提示词（不要什么）

```
用 wan2.7 画一组可爱的小猫组图，4 张，竖版

小猫在花园里玩耍的四个场景。

不要模糊、不要变形、不要多余的手指，画风要干净清新。
```

**Skill 的处理**：
- "不要模糊、不要变形" → 转为 negative_prompt
- wan2.7 支持负向提示词，z-image-turbo 不支持

### 组图的关键注意事项

| 要点 | 说明 |
|------|------|
| **保持主角一致** | 描述时强调"同一只猫"、"同一个角色" |
| **描述场景差异** | 每张图的内容要有明显区别，否则看起来像重复 |
| **明确顺序** | 用 "第一张…第二张…" 或 "首先…然后…" 来暗示顺序 |
| **指定张数** | 虽然说 "一组" Skill 会默认 4 张，但明确说 "3 张" 更可控 |
| **指定尺寸** | "竖版"/"横版"/"方形" 影响最终图片比例 |
| **风格统一** | 在描述开头或结尾强调画风（如"绘本风"、"写实风"） |

### 组图 vs 多张图片的区别

| 说法 | 结果 |
|------|------|
| "画 3 张小猫的图片" | 3 张**独立的**图片，风格可能不一致 |
| "画一组小猫的组图，3 张" | 3 张**风格一致的**连续图片 |
| "画小猫的连环画，3 张" | 同上 |
| "画一个小猫的故事" | 自动编排场景的顺序组图 |

## 图生图 & 多图融合

### 图生图

提供一张参考图片 + 描述，让 AI 基于图片生成新图：

```
把这张图片变成赛博朋克风格
[附上图片 URL 或路径]
```

```
帮我基于这张照片画一幅油画
https://example.com/photo.jpg
```

**注意**：
- wan2.7 和 qwen-image-2.0 支持图生图
- z-image-turbo **不支持**图生图（Skill 会自动排除）
- 图片大小限制：qwen < 10MB，wan < 20MB（超限时 Skill 自动压缩）

### 多图融合

提供多张图片，让 AI 融合生成新图：

```
把这张猫和这张狗融合在一起，画出它们一起玩的样子
图片1: cat.jpg
图片2: dog.jpg
```

**注意**：仅 qwen-image-2.0 支持多图融合。

### 图片编辑

基于已有图片进行局部修改：

```
把这张照片的背景换成海滩
https://example.com/photo.jpg
```

## 用户常踩的坑

### 坑 1：说"组图"但得到风格不一致的图片

**原因**：没有触发 wan2.7 的顺序生成功能。

**解决**：确保提示词中包含 **"组图"、"连环画"、"故事"** 等关键词，或直接指定 "用 wan2.7"。

### 坑 2：让 z-image-turbo 做图生图

**原因**：z-image-turbo 不支持图生图和图片编辑。

**解决**：Skill 会自动排除。如果你手动指定 z-image-turbo 做图生图，Skill 会提示不支持。

### 坑 3：提示词太长被截断

**原因**：有些模型对提示词长度有限制。

**建议**：提示词控制在 100-200 字之间，关键信息放在前面。

### 坑 4：图片保存位置找不到

**现状**：图片优先保存到当前工作目录的 `output/` 文件夹下。如果当前目录没有写权限（如系统根目录），会保存到 `~/Pictures/qwen-image-generator/`（macOS/Linux）或 `C:\Users\<用户名>\Pictures\qwen-image-generator\`（Windows）。

**建议**：生成后注意看 Skill 输出的保存路径。

### 坑 5：Seed 不能复现相同图片

**原因**：三个模型都接受 `seed` 参数，但**均不保证复现性**。

**建议**：如果满意某张图，直接保存文件，不要指望用相同 seed 重新生成。

### 坑 6：API 报错后没有降级

**原因**：三个模型共用同一个 API Key，一个挂了其他也挂了，不做降级。

**建议**：检查 API Key 是否过期，或百炼平台是否有故障。

## FAQ

### Q: 我不懂 AI 生图，能直接用吗？
**A: 可以。** 直接说 "帮我画一只猫" 就行，Skill 会自动处理一切。

### Q: 哪个模型最好？
**A: 各有擅长。**
- 综合最强 → qwen-image-2.0（默认选择）
- 组图/连环画 → wan2.7
- 快速电影感 → z-image-turbo
- 文字/信息图 → qwen-image-2.0
- 不确定 → 让 Skill 自动推荐

### Q: 能生成 4K 图片吗？
**A: 不能。** 三个模型最高只支持 2K（2048×2048）。如需更高分辨率，需要后期使用 AI 放大工具。

### Q: 能生成视频吗？
**A: 不能。** 本 Skill 只支持静态图片生成。

### Q: 一次最多生成几张？
**A:**
- wan2.7: 单次 1-4 张
- qwen-image-2.0: 单次 1-6 张
- z-image-turbo: 单次 1-4 张

### Q: 图片会保存多久？
**A: 永久保存在本地。** 优先保存到当前工作目录的 `output/` 下；如果不可写，保底保存到 `~/Pictures/qwen-image-generator/`。

### Q: 可以指定用中文还是英文提示词吗？
**A: 不需要。** Skill 会根据模型自动转换：
- wan2.7 → 自然中文
- qwen-image-2.0 → 结构化中文
- z-image-turbo → 英文 + 电影感关键词

你用任何语言描述都可以，Skill 会自动适配。

### Q: 生成的图片质量不好怎么办？
**A:**
1. 丰富提示词：加入更多细节（光线、风格、构图）
2. 换个模型：说 "换个模型试试"
3. 重新生成：说 "不满意，重新生成"
4. 加负向提示词：说 "不要模糊、不要变形"
