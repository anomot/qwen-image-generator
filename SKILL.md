# Qwen Image Generator

智能 AI 生图工具，集成阿里云百炼三个主流生图模型，自动选择最佳模型并优化提示词。

## 核心工作流程

```
用户请求 → 意图理解 → 模型推荐 → 提示词处理 → 图片处理 → 生成 → 保存
```

### 1. 意图理解（Agent 自身模型）

分析用户请求，提取：
- **任务类型**：文生图 / 图生图 / 多图融合 / 图片编辑
- **场景描述**：主体、环境、氛围
- **风格偏好**：写实 / 插画 / 卡通 / 电影感
- **尺寸需求**：竖版海报 / 横版封面 / 方形图标
- **输入图片**：URL 或本地路径（图生图时）

### 2. 模型推荐

根据场景自动选择最佳模型：

> **Qwen 系列两个版本**：
> - `qwen-image-2.0`（加速版）：效果与性能平衡，日常使用首选
> - `qwen-image-2.0-pro`（满血版，ID: `qwen-image-2.0-pro-2026-06-22`）：系列最强，文字渲染 + 真实质感最佳，生成较慢

| 触发条件 | 推荐模型 | 理由 |
|---------|---------|------|
| "文字"/"排版"/"信息图"/"中文"/"PPT"/"海报" | **qwen-image-2.0-pro** (满血版) | 文字渲染最强，排版最精确 |
| "写实照片"/"真实感"/"高清细节"/"摄影" | **qwen-image-2.0-pro** (满血版) | 真实质感最细腻 |
| "组图"/"连环画"/"故事系列" | **wan2.7** | 支持顺序生成，风格一致 |
| "快速"/"电影感"/"胶片风" | **z-image-turbo** | 速度最快，电影感强 |
| 需要图生图/图片编辑（质量优先） | **qwen-image-2.0-pro** (满血版) | 编辑细节更好 |
| 需要图生图/图片编辑（速度优先） | **qwen-image-2.0** (加速版) | 速度快，够用 |
| 需要多图融合（速度优先） | **qwen-image-2.0** (加速版) | 速度快 |
| 需要多图融合（质量优先） | **qwen-image-2.0-pro** (满血版) | 融合细节更好 |
| 无特殊要求 | **qwen-image-2.0** (加速版) | 日常使用，速度效果平衡 |

**⚠️ 模型选择优先级（严格遵守，违反会选错模型）**：

> **用户明确指定模型 → 直接使用，不要用提示词关键词重新匹配**

**第一优先级：用户显式指定模型（必须遵守）**
- 用户说"qwen 最强"/"qwen pro"/"满血版"/"最高质量"/"最好效果" → `qwen-image-2.0-pro`
- 用户说"qwen"/"默认"/"正常生成" → `qwen-image-2.0`
- 用户说"wan"/"万相" → `wan2.7`
- 用户说"z-image"/"turbo"/"快速" → `z-image-turbo`

**第二优先级：仅当用户未指定模型时，才根据场景关键词自动推荐**
- 根据上方"触发条件"表格中的关键词选择模型
- 注意：即使用户指定了 pro 模型，提示词中包含"文字"/"排版"等关键词时，**仍然用 pro**，不要降级到加速版

**错误示范（必须避免）**：
```
用户: "使用 qwen 最强模型，帮我做一张知识卡片"
错误: 检测到"知识"关键词 → 推荐 qwen-image-2.0（加速版）❌
正确: 用户已指定"qwen 最强" → 使用 qwen-image-2.0-pro ✅
```

**排除规则**：
- 图生图/图片编辑任务 → 排除 z-image-turbo
- 多图融合任务 → 只能用 qwen-image-2.0 系列

### 3. 提示词处理（三阶段）

#### 阶段一：诊断与增强

使用 Agent 自身模型评估提示词完整度（0-100）：

**评估维度**：
- 画面主体是否清晰
- 场景/环境是否描述
- 光线/氛围是否指定
- 风格/调性是否明确
- 构图/视角是否说明
- **视觉焦点是否明确**（是否有主体突出/虚实对比/视觉引导）

**处理逻辑**：
- 用户说"不要优化"/"直接使用" → 跳过所有增强（包括锚点）
- 完整度 ≥ 80 → 仅做风格适配 + 锚点增强
- 完整度 < 80 → 增强 + 风格适配 + 锚点增强
- 检测到矛盾 → 询问用户

#### 阶段二：视觉锚点增强

很多生成的图片效果不错但缺乏"抓眼球"的地方。视觉锚点（focal point）能让画面有主次层次，而非所有元素平均分布。

**核心原则：只加构图/镜头语言，不加具体内容。增强的是"层次"而非"改变画面"。**

##### 跳过锚点的条件（满足任一即跳过）

| 条件 | 原因 |
|------|------|
| 用户提示词已含锚点语言（特写/近景/微距/背景虚化/景深/bokeh/主体/焦点/视觉中心/前景/聚光灯/逆光/轮廓光） | 用户已自己处理焦点 |
| 任务类型是图生图/图片编辑 | 输入图片已有构图，注入新锚点会冲突 |
| 任务类型是多图融合 | 焦点应由融合指令决定 |
| 任务类型是组图/连环画（wan story） | 每帧焦点不同，统一锚点破坏叙事节奏 |
| 检测到对称/对比/全景/纹理/等距关键词 | 这些场景刻意追求平等权重或无传统焦点 |
| 用户说"不要优化" | 尊重用户原意 |
| 用户明确要求无焦点/平等展示（"不要特写"/"所有元素平等"/"均构图"） | 尊重用户意图 |
| 检测到抽象/超现实/极简/几何/图案/万花筒关键词 | 这些风格刻意打破传统焦点规则 |

##### 未跳过时：判断主焦点

多元素画面需要确定"谁是主角"，建立主→次→背景的三层结构：

**优先级 1：用户显式类型关键词覆盖**

| 用户关键词 | 主角锁定 | 示例 |
|-----------|---------|------|
| "产品图"/"广告"/"商品" | 产品 | "产品图：女孩手持果酒" → 果酒是焦点 |
| "人像"/"肖像"/"角色设计" | 人物 | "人像：女孩在果园里" → 女孩是焦点 |
| "风景"/"场景"/"环境" | 场景 | "海边风景，远处有个女孩" → 海景是焦点 |
| "信息图"/"海报"/"卡片" | 核心信息/标题 | "活动海报，背景是海边" → 标题文字是焦点 |

**优先级 2：语法主语或第一个详细描述的对象**

| 提示词 | 主角推断 | 依据 |
|--------|---------|------|
| "一个女孩拿着一瓶果酒站在海边" | 女孩 | 语法主语，最先描述 |
| "一瓶果酒被女孩拿在海边" | 果酒 | 语法主语变了 |
| "海边，女孩拿着果酒" | 女孩 | 第一个被描述的对象 |

**优先级 3：描述详细度** — 哪个对象被描述得最详细，哪个就是主角

##### 注入视觉层级

确定主角后，建立三层结构（不是"只突出主角"，而是让画面有主次之分）：

| 层级 | 角色 | 锚点语言 |
|------|------|---------|
| **主焦点** | 用户推断的主角 | 主体清晰，光线聚焦，居中或黄金分割位 |
| **次要元素** | 配角/道具 | 适度虚化，作为陪衬 |
| **背景层** | 环境/远景 | 柔和虚化，营造氛围 |

**信息图/文字排版例外**：不用镜头虚化语言，改用信息层级——"标题为视觉焦点，配色对比突出核心信息，辅助信息弱化处理"。

**提示词过长（>200字）例外**：只注入简短锚点（"主体突出，背景虚化"），不展开完整三层结构。

##### 注入强度

| 画面元素数 | 强度 | 注入方式 | 示例 |
|-----------|------|---------|------|
| 单元素（只有一个描述对象） | **简短** | 只加"主体突出，背景虚化" | "一只猫趴在窗台上" → "...主体清晰突出，窗台背景柔和" |
| 多元素（≥2 个描述对象） | **完整三层** | 主焦点/次要元素/背景层 | "女孩拿果酒在海边" → 三层结构 |

##### 按模型风格适配锚点

| 模型 | 锚点融入方式 | 示例（"女孩拿着果酒在海边"） |
|------|------------|--------------------------|
| **wan2.7** | 自然中文，像聊天补充 | "...夕阳西下，女孩为画面视觉焦点，面部光线柔和，果酒作为前景道具适度清晰，海面背景柔和虚化" |
| **qwen 加速版** | 结构化中文，加"焦点"要素 | "...；焦点：女孩居中，果酒中景，海面背景虚化；风格：写实" |
| **qwen 满血版** | 结构化精细焦点描述 | "...；焦点：女孩面部为视觉中心，眼神光突出，果酒前景中景清晰，海面浅景深虚化；风格：超写实" |
| **z-image-turbo** | 英文镜头语言 | "..., sharp focus on the girl's face, wine bottle in midground, ocean blurred in background, shallow depth of field, bokeh" |

##### 多元素锚点增强示例

**用户提示词**：一瓶果酒放在木质桌面上，旁边有新鲜水果和绿叶，远处是厨房

**推断**：果酒是主角（位置核心+"放在桌上"），水果绿叶是配角（"旁边"），厨房是背景（"远处"）

| 模型 | 增强后提示词 |
|------|------------|
| wan2.7 | 一瓶果酒放在木质桌面上，旁边有新鲜水果和绿叶，远处是厨房，**果酒瓶为画面视觉焦点，瓶身标签清晰，水果绿叶作为陪衬适度虚化，厨房背景柔和虚化** |
| qwen-pro | 画面主体：果酒瓶；场景：木质桌面，水果绿叶，厨房；**焦点：果酒瓶居中突出，浅景深虚化背景，定向光突出瓶身质感**；风格：商业摄影 |
| z-image | a bottle of fruit wine on wooden table, fruits and leaves nearby, kitchen in background, **sharp focus on the bottle, shallow depth of field, bokeh background**, commercial photography |

##### 传递给 CLI（重要）

锚点增强完成后，Agent 必须将结果传递给脚本的 `--focal-point` 参数，以便记录到 JSON sidecar：

| 情况 | `--focal-point` 值 | 示例 |
|------|-------------------|------|
| 注入了锚点 | 注入的锚点描述文本 | `"女孩为视觉焦点，果酒中景，海面虚化"` |
| 跳过锚点（图生图/编辑） | `"skipped:img2img"` | — |
| 跳过锚点（多图融合） | `"skipped:multi_fusion"` | — |
| 跳过锚点（组图/连环画） | `"skipped:story"` | — |
| 跳过锚点（用户已有锚点语言） | `"skipped:user_has_anchor"` | — |
| 跳过锚点（对称/全景/纹理等） | `"skipped:no_traditional_focus"` | — |
| 跳过锚点（抽象/极简等风格） | `"skipped:abstract_style"` | — |
| 跳过锚点（用户要求无焦点） | `"skipped:user_no_focus"` | — |
| 用户说"不要优化" | **不传** `--focal-point` | 锚点和提示词优化全部跳过 |

> **注意**：`--focal-point` 仅用于 sidecar 记录，不影响 API 调用和图片生成。

#### 阶段三：风格适配

根据目标模型调整提示词风格（在锚点增强之后进行）：

| 模型 | 风格要求 | 示例 |
|------|---------|------|
| **wan2.7** | 自然中文，像跟朋友说话 | "画一只橘猫趴在窗台上晒太阳，阳光暖暖的洒下来" |
| **qwen-image-2.0** (加速版) | 结构化中文，分要素描述 | "画面主体：橘猫；姿态：趴在窗台；场景：阳光充足；光线：温暖午后；风格：治愈系" |
| **qwen-image-2.0-pro** (满血版) | 结构化中文，更精细的细节描述 | "画面主体：橘猫，毛发细腻；姿态：慵懒趴着；场景：阳光充足，光影斑驳；光线：温暖午后金色光；风格：超写实，高清毛发细节" |
| **z-image-turbo** | 英文 + 电影感关键词 | "film grain, cinematic lighting, Kodak Portra 400, a tabby cat on windowsill, warm golden hour" |

**电影感关键词**：film grain, cinematic lighting, shallow depth of field, bokeh background, analog film texture, Kodak Portra 400

### 4. 图片处理（图生图时）

#### 输入处理

| 输入类型 | 处理方式 |
|---------|---------|
| HTTP/HTTPS URL | **直接传给 API**，不下载 |
| 本地文件 | 转 base64 data URI |
| 文件超限 | 自动压缩（JPEG q85 → 缩小分辨率） |

**文件大小限制**：
- wan2.7: ~20MB
- qwen-image-2.0: 10MB
- z-image-turbo: 不支持图生图

#### 压缩策略

```python
# 1. 先尝试降低 JPEG 质量（q85 → q70 → q50 → q30）
# 2. 仍超限则缩小分辨率到 2048×2048 以内
# 3. 压缩后告知用户：原始大小 → 压缩后大小
```

### 5. 智能分辨率推荐

**重要**：必须根据用户提到的场景关键词选择对应的 `size` 参数值（如 `"portrait"`），而不是直接传入像素值。

| 场景关键词 | 推荐比例 | size 参数值 | 实际像素 |
|-----------|---------|------------|---------|
| "海报"/"朋友圈"/"手机壁纸"/"竖版" | 竖版 9:16 | `"portrait"` | wan: 768×1152, qwen: 1152×2048, z-image: 1120×1440 |
| "公众号封面"/"横幅"/"横版" | 横版 16:9 | `"landscape"` | wan: 1152×768, qwen: 2048×1152, z-image: 1440×1120 |
| "头像"/"图标"/"方形" | 方形 1:1 | `"1k"` 或 `"square"` | 1024×1024 |
| "高清"/"2K"/无明确偏好 | 最大方形 | `"2k"` | wan: 2048×2048, qwen: 2048×2048, z-image: 1440×1440 |

**调用示例**：
```bash
# 用户说"竖版海报" → 使用 size="portrait"
python scripts/qwen_image_gen.py text2img "提示词" --size portrait

# 用户说"横版封面" → 使用 size="landscape"  
python scripts/wan_image_gen.py text2img "提示词" --size landscape

# 无特殊要求 → 使用 size="2k" (默认最大方形，推荐)
python scripts/z_image_gen.py text2img "提示词" --size 2k
```

**注意**：wan2.7 像素总量可超 4K（4096×4096），但无标准 4K 比例（3840×2160）。

### 6. 生成与保存

#### 调用脚本

**基本格式**：
```bash
# wan2.7
python scripts/wan_image_gen.py text2img "提示词" --model [pro/std] --size [size_value] --n [数量] --filename-prefix "[命名前缀]" --focal-point "[锚点描述]"

# qwen-image-2.0
python scripts/qwen_image_gen.py text2img "提示词" --model [default/pro] --size [size_value] --n [数量] --filename-prefix "[命名前缀]" --focal-point "[锚点描述]"

# z-image-turbo
python scripts/z_image_gen.py text2img "提示词" --size [size_value] --n [数量] --filename-prefix "[命名前缀]" --focal-point "[锚点描述]"
```

> **`--focal-point` 使用规则**：
> - **传入**：当 Agent 做了锚点增强且未跳过时，将锚点描述传入（如 `"女孩为视觉焦点，果酒中景，海面虚化"`）
> - **传入跳过原因**：当锚点被跳过时，传入跳过原因（如 `"skipped:img2img"` / `"skipped:user_has_anchor"` / `"skipped:abstract_style"`）
> - **不传**：当用户说"不要优化"/"直接使用"时，不传 `--focal-point`，也不注入锚点到提示词

**完整示例**：
```bash
# 示例1：用户要求"生成一张竖版春日海报，要有花朵和阳光"
# → 推荐 qwen-image-2.0 加速版（无特殊需求），竖版用 portrait
# → 锚点增强：花朵为视觉焦点（单元素简短锚点）
python scripts/qwen_image_gen.py text2img   "画面主体：春日花园；场景：花朵盛开，阳光明媚；光线：温暖的自然光；风格：清新治愈"   --model default --size portrait --n 1 --filename-prefix "春日海报_花朵阳光_清新治愈"   --focal-point "花朵为视觉焦点，阳光营造温暖氛围"

# 示例2：用户要求"用 wan2.7 生成一组小猫日常的连环画"
# → 使用 wan2.7，组图用 n=3，启用顺序生成
# → 组图跳过锚点（每帧焦点不同）
python scripts/wan_image_gen.py text2img   "一只小橘猫的日常：早上起床、中午玩耍、晚上睡觉"   --model pro --size landscape --n 3 --enable-sequential --filename-prefix "小猫日常_连环画"   --focal-point "skipped:story"

# 示例3：用户要求"快速生成一张电影感的城市夜景"
# → 使用 z-image-turbo（快速+电影感），英文提示词
# → 锚点增强：城市天际线为视觉焦点
python scripts/z_image_gen.py text2img   "film grain, cinematic lighting, cyberpunk city at night, neon lights, sharp focus on skyline, shallow depth of field"   --size hd --n 1 --filename-prefix "城市夜景_赛博朋克_电影感"   --focal-point "sharp focus on skyline, neon lights bokeh background"
```

#### 图片命名

使用 `filename_prefix` 参数传入命名前缀，格式：`主体_场景_风格`

**命名规则**：
- 从提示词提取关键词作为文件名前缀（中文或英文均可）
- 用下划线 `_` 分隔不同要素
- wan2.7 组图时自动添加 `_seq01`、`_seq02` 等序列号

**调用示例**：
```bash
# 单张图片
python scripts/qwen_image_gen.py text2img "一只可爱的橘猫趴在窗台上"   --model default --filename-prefix "可爱橘猫_窗台_温暖治愈" --focal-point "橘猫为视觉焦点，窗台背景虚化"

# 生成文件：可爱橘猫_窗台_温暖治愈_20260704_120000.png

# wan2.7 组图（自动加序列号）
python scripts/wan_image_gen.py text2img "小猫的一天" --model pro --n 3   --filename-prefix "小猫日常"

# 生成文件：
# 小猫日常_20260704_120000_seq01.png
# 小猫日常_20260704_120000_seq02.png  
# 小猫日常_20260704_120000_seq03.png
```

**注意事项**：
- 文件名前缀建议控制在 20 字以内
- 避免使用特殊字符（如 `/`、`\`、`:`）
- 如果不传 `filename_prefix`，则使用默认的任务类型（如 `text2img`）

#### 保存位置

脚本自动将图片保存到 **日期子目录**：`output/YYYY-MM-DD/filename.png`

**优先级**：
1. 当前工作目录下的 `output/YYYY-MM-DD/` 目录（自动创建日期子目录）
2. `~/Pictures/qwen-image-generator/YYYY-MM-DD/`（保底，跨平台统一位置）

> macOS: `~/Pictures/qwen-image-generator/YYYY-MM-DD/`
> Windows: `C:\Users\<用户名>\Pictures\qwen-image-generator\YYYY-MM-DD\`
> Linux: `~/Pictures/qwen-image-generator/YYYY-MM-DD/`

**⚠️ Agent 必须遵守的保存规则（违反会导致文件丢失）**：
1. **不要自己下载图片**：脚本已经完成了下载和保存，Agent 绝不需要再下载或保存图片
2. **不要自己创建文件**：Agent 不应使用 `exec_command` 的 `curl`、`wget` 或其他工具下载图片
3. **直接使用脚本输出路径**：读取脚本输出的 `✅ 已保存: <路径>` 行，用这个路径展示图片给用户
4. **不要复制到 output/**：图片已在日期子目录中，不需要额外移动
5. **如果脚本报错**：展示错误信息给用户，不要尝试"自己下载"作为 fallback

## 生成元数据

每张图片生成后会自动保存一个 `.json` sidecar 文件，记录生成参数：

```json
{
  "image": "可爱橘猫_窗台_20260704_222226.png",
  "model": "wan2.7-image-pro",
  "prompt": "画一只橘猫趴在窗台上，阳光透过玻璃...",
  "size": "1024*1024",
  "n": 1,
  "seed": 42,
  "negative_prompt": "模糊，低质量",
  "focal_point": "橘猫为视觉焦点，毛发清晰，窗台背景柔和虚化",
  "generated_at": "2026-07-04T22:22:27.255399",
  "api_image_url": "https://dashscope-7c2c.oss-accelerate.aliyuncs.com/..."
}
```

**focal_point 字段说明**：
- 注入了锚点：记录注入的锚点语言（如 `"橘猫为视觉焦点，背景虚化"`）
- 跳过锚点：记录跳过原因（如 `"skipped:img2img"` / `"skipped:user_has_anchor"` / `"skipped:abstract_style"`）
- 旧版兼容：不出现此字段

**用途**：
- 回溯上次生成参数，方便微调
- 复现相同的图片效果
- 批量管理生成历史
- 追溯锚点处理策略，方便"再生成一张类似的"

## 健壮性特性

- **下载重试**：图片下载失败时自动重试 3 次（指数退避）
- **文件名校验**：`filename_prefix` 自动清理非法字符（`/`, `:`, `\` 等替换为 `_`）
- **响应兼容性**：同时支持 SDK 对象响应和字典响应两种格式

## Seed 参数说明

- 所有模型都接受 `seed` 参数
- **均不保证复现性**：相同 seed + 相同提示词会产生不同图片
- 如需复现，需保存生成的图片文件本身
- 文件命名时可包含 seed 值便于追溯

### 技术错误

| 错误类型 | 处理方式 |
